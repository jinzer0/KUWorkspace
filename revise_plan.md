
수정 리스트
1. 초기 운영 시점을 한 번만 입력하고 재시작 후에 다시 묻지 않기
- data/clock.txt에 0000-00-00T00:00 초기 저장
- 프로그램 시작시 clock.txt에 저장된 시점이 0000-00-00T00:00이면 시점 입력, 아니면 저장된 시점으로 프로그램 시작

2. 시점 이동한 유저에게 모든 책임을 부여하기
- 유저가 다른 유저, 관리자의 필수 이벤트 처리를 무시하고 시점을 이동하려한 경우, 경고 메세지를 보여주고, 시점 이동을 강행하면, 그로 인해 발생하는 모든 패널티를 시점 이동한 유저에게 자동 부과하기

3. 유저 회의실 예약 취소 절차 변경
- 회의실 예약 취소 메뉴 진입 -> 취소 예약 선택 -> 취소 여부 확인(조건에 맞는 경우 패널티 안내) -> 취소 확정 -> 취소 처리 (조건에 맞는 경우 패널티 부과)

4. final_plan에 없는 추가 기능 삭제
- 현재 코드에는 진행 중 예약 재배정/교체 흐름이 있습니다. src/cli/admin_menu.py, src/domain/room_service.py, src/domain/equipment_service.py, 그리고 관련 테스트들(tests/unit/test_admin_menu.py, tests/e2e/test_admin_scenarios.py)이 그 증거

5. 패널티 no_show 삭제
- 패널티 no_show를 구현하지 않기로 했으므로 삭제

6. 패널티 자동 부과하기
- 기존 관리자의 패널티 수동 부과 기능은 유지
- 직전 취소, 회의실 퇴실 지연, 장비 반납 지연 패널티는 기한 운영 시점내 유저가 처리하지 않고 해당 기한 시점 이후로 이동시 자동으로 부과되는 로직 추가
