# **공유 오피스 예약 및 장비 대여 관리 프로그램**

1차 기획서 원판  

# **1\. 용어**

\- 프로그램: 이 문서에서 명세하는 대상 CLI 프로그램 전체를 의미한다.  
\- 게스트: 로그인하지 않은 사용자다.  
\- 유저: "user" 역할을 가진 로그인 사용자다.  
\- 관리자: "admin" 역할을 가진 로그인 사용자다.  
\- 사용자: 게스트, 유저, 관리자를 모두 지칭한다.  
\- 엔터키: 사용자의 입력을 종료하고 다음 동작(예: 메뉴 실행, 상태 전이 등)을 유발하는 키를 의미한다.  
\- 입력: 사용자가 키보드를 통해 문자열을 구성하고, 엔터키를 통해 해당 입력을 확정하거나 입력을 종료하는 일련의 행위를 의미한다.  
\- 공백류: 탭문자,  개행문자,  스페이스 등 모든 공백 문자를 포함한다 .  
\- 운영 시점: 시스템에서 사용하는 시간 단위로, 09:00과 18:00 두 시점만 존재하며 모든 예약 및 이벤트 처리는 이 시점을 기준으로 이루어진다.  
\- 현재 운영 시점: 현재 시스템에서 사용 중인 운영 시점이다.  
\- 시점 전환: 현재 운영 시점에서 다음 운영 시점으로 이동하는 과정으로, 이 과정에서 예약 시작 및 종료 처리, 패널티 적용 등의 시스템 이벤트가 함께 수행된다.  
\- 패널티 점수: 사용자 행동에 따라 부여되는 점수로, 누적 점수에 따라 사용자 상태가 변경된다.  
\- 사용자 상태: 패널티 점수에 따라 결정되는 사용자 이용 상태로, 정상(normal), 제한(restricted), 이용금지(banned)로 구분된다.  
\- 조기 퇴실(반납): 유저가 예약된 대여 종료 시점 보다 앞서 회의실(또는 장비)을 관리자에게 반납하고 이용을 종료하는 행위를 지칭한다.  
\- 메뉴: 메뉴(menu)는 특정 사용자 상태 또는 권한에서 접근 가능한 기능 목록 또는 화면 진입 항목의 집합으로, 크게 비로그인 메뉴, 유저 메뉴, 관리자 메뉴로 구분된다. 유저 메뉴 및 관리자 메뉴는 로그인한 사용자에게 제공되며, 권한에 따라 유저 메뉴 및 관리자 메뉴로 구분된다.  
\-점검기간: 회의실 또는 장비 상태가 \[점검중\]으로 바뀐 시점에서, 기존 퇴실 또는 반납 종료일 기준 다음날 09시까지를 뜻한다.

# **2\. 기본사항**

# **2.1 작동 환경**

프로그램은 Python 3.13 버전이상의 .py 파일을 실행시킬 수 있는 환경에서 제대로 동작하며 그 외 환경에서는 작동이 보장되지 않는다.

# **2.2 프로그램 구성**

진입점은 main.py이며 그 외 docs/, domain/, data/, scripts/, src/로 구성되어 있다.

프로그램의 올바른 실행 방법은 shell에 python main.py를 통해 진입한다.   
모든 데이터 파일의 경로는 ./data/{fileName}.txt이다. 이 파일은 사용자가 직접 수동으로 이동하거나 삭제하기 전 까지는 프로그램을 통해 삭제하거나 이동하지 않는다.

# **2.3 프로그램 설치 및 실행**

기본적으로 아무 경로에나 프로그램 구성 파일을 복사하여 설치한다. 단, 해당 경로에는 어떠한 다른 파일도 없어야 한다(빈 디렉토리여야 한다).

## **4.1.1 아이디**

문법 형식: 아래 조건을 모두 만족하는 문자열이다.

- 문자 기준 길이가 3 이상 20이하  
- 숫자(0-9) 또는 알파벳 대소문자, “\_”로 구성  
- 공백 미포함

의미 규칙: 유일성 보장을 위해 다른  계정의 아이디와 중복될 수 없다. 사용자 로그인시 사용된다.  
동치 비교: 회원가입시 입력하는 아이디와 로그인시 입력하는 아이디의 동치 비교는 각  문자의 순서와 내용이 정확히 일치해야 한다. 중복 여부를 검사하는 경우,  대소문자 는 서로 다른   문자로 간주한다.  
다음 나열된 두 쌍의 아이디들은 서로 동치가 아니다.  
Computer, computer  
oFFice, ofFice

- 문자 기준 길이가 4 이상 50 이하  
- 공백 미포함

의미 규칙: 사용자 로그인 인증에 사용된다.   
동치 비교: 두 비밀번호를 비교할 경우 모든 문자의 순서와 내용이 정확히 일치해야 한다. 다음 나열된 두 쌍의 비밀번호들은 서로 동치가 아니다.

# **4.2 날짜 및 시간**

날짜 및 시간은 프로그램 가상 시점부터 유저의 회의실/장비 예약 시작 시각, 종료 시각, 생성 시각, 수정 시각 등을 표현한다.

## **4.2.1 날짜**

예약 생성 및 변경, 프로그램 가상 시점 설정 과정에서 사용자로부터 날짜를 입력 받는다.

문법 규칙: 다음의 조건을 모두 만족시키는 문자열이다.

- YYYY-MM-DD, YYYY.MM.DD, YYYY MM DD 형식  
- 연도 Y는 2026\~2100 사이의 정수, 월은 1\~12 사이의 정수, 일은 1\~31 사이의 정수로 구성  
- 각 부분 구분자는 “-”, “.”, “ “로 형성되어야 하며 한 문자열에 여러 구분자를 혼합해 형성할 수 없음  
- 월, 일은 실제 숫자 자리수와 상관 없이 필요 자리수보다 적은 경우 앞에 0을 붙여야함(2026.4.3은 2026.04.03으로 입력)  
- 문자열 선후 공백 미포함

의미 규칙: 

- 일은 실제 해당 월에 존재하는 날짜이어야 한다. 예시로 2월은 31일이 존재하지 않음으로 입력할 수 없다.

## **4.2.2 시간**

문법 규칙: 다음의 조건을 모두 만족시키는 문자열이다.

- HH:MM, HHMM 형식  
- 시는 09 또는 18, 분은 00로 구성  
- 구분자는 “:”를 사용하거나 구분자 없이 형성되며 혼합해 사용할 수 없음  
- 공백 미포함  
- 생성 시각과 수정 시각 등 프로그램 예약 처리 과정에 영향을 주지 않는 시간은 시, 분을 실존하는 시각으로 사용 가능

의미 규칙:

- 실존하는 시각으로 해석 가능한 값

# **5\. 데이터 파일**

프로그램 실행 중에 사용자에게 입력을 받거나 미리 정의된 정보를 통해 data 폴더내 .txt 텍스트 파일로 존재한다. 프로그램 실행 중에 사용자의 동작이나 입력에 의해 이러한 데이터 파일을 읽고 쓴다. 프로그램 실행시 필수 데이터 파일이 data 폴더 내 존재하지 않는 경우 필수 데이터 파일을 생성한다. 프로그램 동작중 사용자가 임의로 텍스트 편집기 등을 이용해 데이터 파일을 수정/삭제를 하는 경우 정상적인 동작을 보장할 수 없다. 프로그램을 통하지 않고 직접 수정할 수 있다. 단 아래의 조건을 만족한다.

- 프로그램이 실행되지 않는 중에 수정  
- 문법과 의미 규칙 준수  
- UTF-8 인코딩 사용  
- Linux, MacOS 운영체제의 개행 방식 사용

# **5.1 문법 규칙**

모든 데이터 파일은 다음 조건을 모두 만족한다.

- 하나의 행이 레코드 1개 표현  
- 필드 구분자로 “|” 사용  
- 필드값에서 “|”, “\\”를 표현해야 할 경우 “\\|”, “\\\\”와 같이 이스케이프  
- 값이 존재하지 않는 경우 “\\-” 사용  
- 값이 빈 문자열인 경우 “||”처럼 구분자 사이에 아무 문자도 저장하지 않음  
- 빈 행 불가  
- 날짜 및 시간 등 여러 형식으로 입력받는 값은 하나의 형식으로 표준화해 저장  
- 날짜는 “YYYY-MM-DD”, 시간은 “HH:MM” 형식으로 표준화해 저장  
- 날짜 및 시간을 한 필드값에 저장시 “YYYY-MM-DDTHH:MM”형식으로 저장

# **5.2 사용자 계정 파일**

사용자 계정은 data/users.txt에 저장하고 읽고 쓴다. 다음과 같은 형식으로 구성되어 있다.  
(화살표 괄호 \<\>은 문법 설명용 기호)  
의미 규칙:

- username: 아이디  
- password: 비밀번호  
- role: 계정 구분 용도, “user”, “admin” 값만 허용  
- penalty\_points: 계정에 누적된 패널티 점수, 0 이상의 정수만 허용  
- normal\_use\_streak: 정상 이용 연속 횟수, 0 이상의 정수만 허용  
- restriction\_until: 이용 제한 종료 시각, 값이 없으면 “\\-”  
- created\_at: 계정 생성 시각  
- updated\_at: 마지막 수정 시각  
- restriction\_until, created\_at, updated\_at은 날짜+시간만 허용

# **5.3 회의실**

## **5.3.1 필드 정의 및 규격**

회의실 파일에 포함되는 데이터 값은 아래의 정의를 따르며, 정확한 형식을 준수해야 한다.

- name: 회의실 이름, 모든 회의실 이름은 중복되지 않음, “회의실” \+ “ “ \+ 1자리 정수(0\~9)+ 알파벳 대문자 1자 형식  
- capacity: 수용 인원, 1 이상의 정수만 허용  
- location: 회의실 위치 정보, 1자리 정수(0\~9) \+ “층” 형식  
- status: 회의실 운영 상태, “available”, “maintenance”, “disabled” 값만 허용  
- description: 회의실 설명 문자열, 문자 기준 1\~10 길이의 문자열(개행 문자 허용 안됨)  
- created\_at: 회의실 등록 시각  
- updated\_at: 마지막 수정 시각  
- created\_at, updated\_at은 날짜+시간만 허용

# **5.4 장비**

## **5.4.1 필드 정의 및 규격**

장비 파일에 포함되는 데이터 값은 아래의 정의를 따르며, 정확한 형식을 준수해야 한다.

- name: 장비 이름, 문자 길이 1\~10 사이의 문자열(공백류 미포함)  
- asset\_type: 장비 종류, 문자 길이 1\~10 사이의 문자열(공백류 미포함)

# **6.3 운영 시계**

### **6.3.1 운영 시계 개요**

### **6.3.1.1 운영 시계 개념**

운영 시계는 본 프로그램에서 사용하는 가상의 시간 관리 시스템으로, 실제 벽시계 시간과는 독립적으로 동작한다.

모든 기능 및 시스템 동작은 운영 시계의 현재 시점을 기준으로 수행되며, 사용자와 관리자는 해당 시점을 기반으로 시스템을 이용하게 된다.

운영 시계는 시스템 전반에서 단일 기준 시계로 사용되며, 모든 사용자와 관리자는 동일한 시점을 공유한다.

이를 통해 시스템 내 시간 불일치 문제를 방지하고, 모든 기능이 동일한 기준에서 일관되게 동작하도록 보장한다.

### **6.3.1.2 시점 구성**

운영 시계는 이산적인 시점을 기반으로 구성되며, 하루는 09:00와 18:00 두 개의 시점으로 이루어진다.

각 시점은 시스템 상태를 구분하는 기준으로 사용되며, 연속적인 시간의 흐름이 아닌 특정 시점 간의 전환을 통해 시간이 표현된다.

운영 시점으로는 09:00와 18:00 두 시점만 허용되며, 이 외의 시각은 사용할 수 없다.

### **6.3.1.3 시점 진행 방식**

시점은 정해진 순서에 따라 순차적으로 진행된다.

현재 시점이 09:00인 경우 당일 18:00으로 이동하며, 현재 시점이 18:00인 경우 다음날 09:00으로 이동한다.  

시점 전환 과정에서 18:00에서 다음 날 09:00으로 전환되는 경우 날짜는 자동으로 다음 날짜로 갱신된다.

### **6.3.1.4 시스템 처리 기준**

모든 사용자 요청 및 시스템 처리 과정은 현재 운영 시계의 시점을 기준으로 수행되며,

동일 시점에 발생한 사용자 요청은 시스템 내부 큐(queue)에 저장되며, 순서에 따라 하나씩 순차적으로 처리된다.

운영시계 메뉴이다.

| \==================================================   운영 시계 \================================================== 현재 운영 시점 : 2100-01-04 09:00 다음 시점 : 2100-01-04 18:00 현재 시점 보기  다음 시점으로 이동 돌아가기 —----------------------------------------------------------------------- |
| :---- |

### **6.3.2 현재 시점 보기**

사용자는 운영 시계 메뉴에서 '현재 시점 보기' 기능을 선택하여 현재 운영 시점과 다음 시점, 그리고 예정 이벤트 정보를 함께 확인할 수 있다.

본 기능은 현재 시스템 상태와 가까운 미래에 발생할 상황을 종합적으로 파악할 수 있도록 하기 위한 조회 기능이다.

시점 정보는 현재 시점과 다음 시점, 총 2개의 시점으로 구성되어 함께 출력된다.

현재 시점은 09:00 또는 18:00 중 하나의 값으로 표현되며, 시스템이 현재 기준으로 삼고 있는 시간 상태를 의미한다.

다음 시점은 현재 시점을 기준으로 결정되어 함께 출력되며, 현재 시점이 09:00인 경우 동일 날짜의 18:00, 현재 시점이 18:00인 경우 다음 날짜의 09:00이 다음 시점으로 출력된다.

시스템은 현재 시점을 기준으로 다음에 발생할 예정인 이벤트 정보를 함께 제공한다.

현재 시점이 09:00인 경우 동일 날짜의 18:00 시점 이벤트를 제공하며,

현재 시점이 18:00인 경우 다음 날짜의 09:00 시점 이벤트를 제공한다.

사용자에게는 본인과 관련된 이벤트만 제공되며, 예약 시작 및 종료, 상태 변경 등의 정보가 포함된다.

관리자에게는 시스템 전체 이벤트가 제공되며, 전체 예약 현황, 패널티 발생 대상, 상태 변경 예정 사용자 등의 정보가 포함된다.

현재 시점 보기 메뉴를 선택 했을 때 다음이 출력된다.

사용자 입장

| \==================================================   운영 시점 정보 \================================================== 현재 운영 시점 : 2100-01-04 09:00 다음 시점 : 2100-01-04 18:00 \[ 예상 이벤트 \] 2100-01-04 18:00로 이동 준비 당일 종료 예정 회의실 0건, 장비 0건 |
| :---- |

관리자 입장

| \==================================================   운영 시점 정보 (관리자) \================================================== 현재 운영 시점 : 2100-01-04 09:00 다음 시점 : 2100-01-04 18:00 \[ 예상 이벤트 \] 2100-01-04 18:00로 이동 준비 회의실 예약 종료 예정 2건 장비 반납 예정 4건 |
| :---- |

### **6.3.3 다음 시점으로 이동**

사용자는 운영 시계 메뉴에서 '다음 시점으로 이동' 기능을 선택하여 현재 시점을 다음 시점으로 전환할 수 있다.

본 기능은 시스템의 시간 흐름을 진행시키는 핵심 기능이다.

현재 시점이 09:00인 경우 당일 18:00으로 전환되며, 현재 시점이 18:00인 경우 다음날 09:00으로 전환된다.

시점 전환이 완료되면 시스템은 변경된 운영 시점을 사용자에게 출력한다.

관리자에 의해 시점이 전환되는 경우, 시점 간 처리 과정에서 발생한 이벤트 결과를 함께 출력하여 전체 시스템 상태 변화를 확인할 수 있도록 한다.

사용자는 시점 전환에 따라 발생한 본인과 관련된 최종 결과만 확인할 수 있다.

다음 시점으로 이동 메뉴를 선택 했을 때 다음이 출력된다.

사용자 입장

| \==================================================   시점 이동 결과  \================================================== 운영 시점을 2100-01-04 18:00로 이동했습니다. \[이벤트\] 본인의 회의실 예약이 종료되었습니다. 본인의 장비 반납 기한이 도래했습니다. |

| \==================================================   시점 이동 결과  \================================================== 운영 시점을 2100-01-04 18:00로 이동했습니다. \[이벤트\] 회의실 예약 종료 처리 2건 장비 반납 기한 도래 4건 |

# **6.5 유저**

## **6.5.1 회의실**

### **6.5.1.0 회의실 예약 데이터 기록 규칙**

회의실 예약의 각 상태 전이 시, 해당 시점(current\_time)을 아래 필드에 기록한다. 기록되지 않은 필드는 "-"로 저장한다.

| 상태 전이 | 기록 필드 |
| ----- | ----- |
| reserved 생성 시 | created\_at, updated\_at |
| reserved → checkin\_requested | requested\_checkin\_at, updated\_at |
| checkin\_requested → checked\_in | checked\_in\_at, updated\_at |
| checked\_in → checkout\_requested | requested\_checkout\_at, updated\_at |
| checkout\_requested → completed | completed\_at, updated\_at |
| reserved → cancelled | cancelled\_at, updated\_at |

직전 취소로 인해 예약이 취소되는 경우 cancelled\_at을 기록하고, penalties.txt에 reason을 late\_cancel로 하여 패널티 레코드를 생성한다.

### **6.5.1.1 회의실 목록 조회**

유저가 로그인 후 메뉴 번호 1을 입력했을 경우, 다음과 같이 회의실 목록을 출력한다. 출력하는 내용은 순서대로 이름, 수용인원, 위치, 상태다.

모든 정보를 출력한 이후 유저가 엔터키를 포함한 문자들을 입력하면 사용자 메뉴로 돌아간다.

| \==================================================   회의실 목록 \================================================== 이름          수용인원  위치    상태 \------------------------------------------------- 회의실 4A      4명      1층     \[사용가능\] 회의실 4B      4명      1층     \[사용가능\] 회의실 4C      4명      1층     \[사용가능\] 회의실 6A      6명      2층     \[사용가능\] 회의실 6B      6명      2층     \[사용가능\] 회의실 6C      6명      2층     \[사용가능\] 회의실 8A      8명      3층     \[사용가능\] 회의실 8B      8명      3층     \[사용가능\] 회의실 8C      8명      3층     \[사용가능\] 계속하려면 Enter를 누르세요... |
| :---- |

상태별 표시 기준은 다음과 같다. available 상태이면 \[사용가능\]을 표시한다. maintenance 상태이면 \[점검중\]을 표시하며 "예약 불가 (문의: 관리자에게 연락하세요)"를 함께 출력한다. disabled 상태이면 \[사용불가\]를 표시하며 동일하게 예약 불가 안내를 출력한다. 등록된 회의실이 없는 경우 "등록된 회의실이 없습니다."를 출력한 후 사용자 메뉴로 돌아간다.

### **6.5.1.2 회의실 예약하기** 

유저가 로그인 후 메뉴 번호 2를 입력했을 경우, 이용 인원, 시작일, 종료일, 회의실 선택 순으로 입력을 받아 예약을 생성한다.

이용금지(banned) 상태인 경우 "이용이 금지된 상태입니다. 해제일: YYYY-MM-DD"를 출력하고 예약하기를 차단한다. 제한(restricted) 상태에서 회의실 활성 예약이 1건 이상인 경우 "패널티로 인해 추가 예약이 불가합니다."를 출력하고 차단한다. 활성 예약(reserved, checkin\_requested, checked\_in, checkout\_requested)이 이미 1건 이상이면 "이미 활성 회의실 예약이 있습니다."를 출력하고 차단한다. 회의실 활성 예약과 장비 활성 예약은 각각 독립적으로 카운트한다.

이용 인원은 1 이상 8 이하의 정수만 허용한다. 정수가 아닌 문자를 입력하면 "숫자를 입력해주세요."를 출력하고 재입력을 받는다. 0 이하의 정수를 입력하면 "1 이상의 인원을 입력해주세요."를 출력하고 재입력을 받는다. 9 이상을 입력하면 "수용 가능한 최대 인원은 8명입니다."를 출력하고 재입력을 받는다. 빈 목록은 표시하지 않는다. q 또는 취소를 입력하면 예약 흐름을 취소하고 메뉴로 복귀한다.

날짜는 YYYY-MM-DD, YYYY.MM.DD, YYYY MM DD 형식만 허용한다. 한 문자열에 구분자를 혼합하여 사용할 수 없다. 연도는 2026 이상 2100 이하의 정수, 월은 01\~12, 일은 01\~31로 구성되며 월과 일은 실제 자릿수보다 적은 경우 앞에 0을 붙여야 한다. 문자열 앞뒤 공백은 허용하지 않는다. 위 형식을 만족하지 않는 경우 "날짜 형식이 올바르지 않습니다. (예: 2099-01-01)"를 출력하고 재입력을 받는다. 입력한 날짜가 실제로 존재하지 않는 날짜인 경우에도 동일하게 출력하고 재입력을 받는다. 시작일이 오늘과 같으면 "당일 예약은 불가합니다. 내일부터 예약 가능합니다."를 출력하고 재입력을 받는다. 시작일이 과거이면 "과거 날짜는 예약할 수 없습니다."를 출력하고 재입력을 받는다. 시작일이 오늘로부터 180일을 초과하면 "예약 시작일은 오늘로부터 180일 이내여야 합니다."를 출력하고 재입력을 받는다. 종료일이 시작일보다 빠르면 "종료일은 시작일보다 빠를 수 없습니다."를 출력하고 재입력을 받는다. 종료일이 시작일로부터 13일을 초과하면 "예약 기간은 최대 14일까지 가능합니다."를 출력하고 재입력을 받는다.

예약 가능한 회의실 목록은 수용 인원 오름차순으로 정렬하여 표시한다. 목록 포함 조건은 상태가 available이고 수용 인원이 이용 인원 이상이며 해당 기간에 충돌하는 활성 예약이 없는 경우다. 해당 인원과 기간에 예약 가능한 회의실이 없으면 "해당 인원과 기간에 예약 가능한 회의실이 없습니다."를 출력하고 메뉴로 복귀한다. 회의실 선택 시 정수가 아닌 문자를 입력하면 "숫자를 입력해주세요."를 출력하고 재입력을 받는다. 목록에 없는 번호를 입력하면 "1\~N 사이의 번호를 입력해주세요."를 출력하고 재입력을 받는다. 더 작은 수용 인원의 회의실이 1개라도 남아있으면 더 큰 회의실 선택 시 "더 작은 회의실이 예약 가능합니다. 해당 회의실을 먼저 이용해주세요."를 출력하고 재입력을 받는다. 선택 후 상태 변경으로 예약이 불가한 경우 "선택한 회의실을 예약할 수 없습니다. 다시 시도해주세요."를 출력한다. 0을 입력하면 메뉴로 복귀한다.

| \================================================== 회의실 예약하기 \================================================== 이용 시간은 매일 09:00 \~ 18:00로 고정됩니다. 예약 시작일은 내일부터 선택할 수 있고, 시작일 기준 최대 180일까지 가능합니다. 예약 기간은 1일 이상 14일 이하입니다. 이용 인원: 4 시작 날짜 (YYYY-MM-DD / YYYY.MM.DD / YYYY MM DD): 2099-03-03 종료 날짜 (YYYY-MM-DD / YYYY.MM.DD / YYYY MM DD): 2099-03-14 1\. 회의실 4A (4명, 1층) 2\. 회의실 4B (4명, 1층) 3\. 회의실 4C (4명, 1층) 4\. 회의실 6A (6명, 2층) 5\. 회의실 6B (6명, 2층) 6\. 회의실 6C (6명, 2층) 7\. 회의실 8A (8명, 3층) 8\. 회의실 8B (8명, 3층) 9\. 회의실 8C (8명, 3층) 0\. 취소 회의실 선택 (번호): 1 ✓ 예약이 완료되었습니다. 시간: 2099-03-03 09:00 \~ 2099-03-14 18:00 계속하려면 Enter를 누르세요... |
| :---- |

예약 완료 시 room\_bookings.txt에 새 레코드를 생성한다. 이때 status는 reserved, checked\_in\_at / requested\_checkin\_at / requested\_checkout\_at / completed\_at / cancelled\_at은 "-"로 저장하고, created\_at과 updated\_at은 current\_time으로 저장한다.

### **6.5.1.3 회의실 예약 조회**

유저가 로그인 후 메뉴 번호 3을 입력했을 경우, 다음과 같이 본인의 전체 회의실 예약 이력을 출력한다. 출력하는 내용은 순서대로 예약 ID(앞 8자리), 회의실 이름, 예약 기간, 상태다. 모든 정보를 출력한 이후 유저가 엔터키를 포함한 문자들을 입력하면 사용자 메뉴로 돌아간다

| \================================================== 내 회의실 예약 \================================================== 회의실              시간                                        상태 \---------------------------------------------------------------- 회의실 4A  2099-03-03 09:00 \~ 2099-03-14 18:00  \[예약됨\] 회의실 4B  2099-01-01 09:00 \~ 2099-01-05 18:00  \[완료\] 회의실 4C  2098-06-01 09:00 \~ 2098-06-01 18:00  \[취소됨\] 계속하려면 Enter를 누르세요...  |
| :---- |

정렬 기준은 start\_time 내림차순으로 최신 예약을 먼저 표시한다. 최대 20건을 표시하며 초과 시 "... 외 N건"을 안내한다. 예약 내역이 없는 경우 "예약 내역이 없습니다."를 출력한 후 사용자 메뉴로 돌아간다. 

### **6.5.1.4 회의실 예약 변경**

유저가 로그인 후 메뉴 번호 4를 입력했을 경우, 다음과 같이 변경 가능한 예약 목록을 출력하고 날짜 변경을 진행한다. 날짜만 변경 가능하며 회의실 자체는 변경할 수 없다. 이용금지(banned) 상태인 경우 진입 시 차단된다.

(y/n/예/아니오): y ✓ 예약이 변경되었습니다. 새 시간: 2099-03-05 09:00 \~ 2099-03-16 18:00 계속하려면 Enter를 누르세요...  |
| :---- |

목록 표시 조건은 본인 소유이며 reserved 상태인 예약만 해당한다. 예약 시작 시점이 현재 운영 시계의 시점보다 이전이거나 동일한 경우 "이미 시작된 예약은 변경할 수 없습니다."를 출력한다. 회의실 자체 변경을 시도하면 "날짜만 변경 가능합니다. 회의실을 변경하려면 기존 예약을 취소 후 새로 예약하세요."를 안내한다. 날짜는 YYYY-MM-DD, YYYY.MM.DD, YYYY MM DD 형식만 허용하며 한 문자열에 구분자를 혼합하여 사용할 수 없다. 연도는 2026 이상 2100 이하의 정수, 월은 01\~12, 일은 01\~31로 구성되며 월과 일은 실제 자릿수보다 적은 경우 앞에 0을 붙여야 한다. 새 날짜가 기존 날짜와 완전히 동일하면 "변경된 내용이 없습니다. 다른 날짜를 입력해주세요."를 출력하고 재입력을 받는다. 새 날짜가 동일 회의실의 다른 활성 예약과 충돌하면 "해당 기간에 이미 예약이 있습니다. 다른 날짜를 선택해주세요."를 출력한다. 날짜 입력 완료 후 "정말 변경하시겠습니까? (y/n)"를 출력한다. y, yes, 예, ㅇ(대소문자 무관)를 입력하면 변경이 완료되고 변경된 날짜를 출력한 후 메뉴로 복귀한다. n, no, 아니오, ㄴ을 입력하면 변경을 철회하며 메뉴로 복귀한다. 그 외 입력 시 "y 또는 n을 입력해주세요."를 출력하고 재입력을 받는다. 변경 가능한 예약이 없는 경우 "변경 가능한 예약이 없습니다. (예약 대기 상태만 변경 가능)"을 출력한 후 사용자 메뉴로 돌아간다. 

### **6.5.1.5 회의실 예약 취소**

유저가 로그인 후 메뉴 번호 5를 입력했을 경우, 다음과 같이 취소 가능한 예약 목록을 출력하고 취소를 진행한다. 이용금지(banned) 상태인 경우 진입 시 차단된다.

| \==================================================   회의실 예약 취소 \==================================================   1\.  회의실 4A \- 2099-03-05 09:00 \~ 2099-03-16 18:00   0\.  취소 취소할 예약 선택 (번호): 1 정말 취소하시겠습니까? (y/n/예/아니오): y ✓ 예약이 취소되었습니다. 계속하려면 Enter를 누르세요...  |
| :---- |

목록 표시 조건은 본인 소유이며 reserved 상태인 예약만 해당한다. start\_time이 current\_time보다 이전이면 "이미 시작된 예약은 취소할 수 없습니다."를 출력한다. start\_time이 current\_time과 같은 경우(당일 09:00)에는 취소는 가능하나 직전 취소 패널티 2점이 부과된다. 이 경우 취소 확인 전에 반드시 "직전 취소로 인해 패널티 2점이 부과됩니다. 그래도 취소하시겠습니까?"를 출력해야 한다. 유저가 n, no, 아니오, ㄴ을 입력하면 취소를 철회하며 패널티는 부과되지 않는다.

취소 확인 입력으로 y, yes, 예, ㅇ(대소문자 무관)를 입력하면 취소가 진행되고, n, no, 아니오, ㄴ을 입력하면 취소를 철회한다. 그 외 입력 시 "y 또는 n을 입력해주세요."를 출력하고 재입력을 받는다. 취소 후 상태는 reserved에서 cancelled로 전이된다. 취소 가능한 예약이 없는 경우 "취소 가능한 예약이 없습니다."를 출력한 후 사용자 메뉴로 돌아간다.

취소 처리 시 해당 예약 레코드의 status를 cancelled로 변경하고 cancelled\_at에 current\_time을 기록하며 updated\_at을 갱신한다. 직전 취소 패널티가 부과되는 경우 penalties.txt에 아래와 같이 레코드를 생성한다.

| 필드 | 값 |
| ----- | ----- |
| reason | late\_cancel |
| points | 2 |
| related\_type | room\_booking |
| related\_id | 해당 예약의 id |
| memo | 빈 문자열 |
| created\_at, updated\_at | current\_time |

### **6.5.1.6 회의실 입실 신청**

예약 번호 선택 후 해당 예약의 start\_time이 current\_time과 정확히 일치하는지 확인한다. 일치하지 않는 경우 "체크인 요청은 예약 당일 09:00 시점에서만 가능합니다."를 출력하고 메뉴로 복귀한다. 일치하는 경우 "체크인 요청하시겠습니까? (y/n)"를 출력한다.

### **6.6.1.1 전체 회의실 예약 조회**

관리자가 로그인 후 관리자 메뉴에서 메뉴 번호 1을 입력하고 엔터키를 누른 경우 모든 회의실에 대한 예약 정보를 출력한다.

‘회의실 예약 정보 현황’은 다음과 같다.  
 \- 사용중: 현재 운영 시점에 유저가 해당 회의실을 사용 중인 경우  
 \- 예약있음: 사용중은 아니지만, 현재 운영 시점 이후에 유효한 예약이 1건 이상 존재하는 경우   
 \- 예약없음: 사용중인 예약도 없고, 현재 운영 시점 이후에 유효한 예약도 존재하지 않는 경우

\<예약있음\> 및 \<사용중\> 현황인 회의실에 대해선 예약일 정보가 출력돼야 한다.  
이때 \<사용중\>인 회의실에 대한 예약일 정보도 포함해서 출력된다.  
\<예약없음\> 현황을 갖는 회의실은 ‘예약일’ 출력에서 ‘X’ 로 표기된다.

| \================================================= 회의실 목록 \================================================= 이름            수용인원           위치          현황         예약일 —--—------------------------------------------------------ 회의실 4A  4명                  1층            예약없음     X 회의실 4B  4명                  1층            예약없음     X 회의실 4C  4명                  1층            예약없음     X 회의실 6A  6명                  2층            예약없음     X 회의실 6B  6명                  2층            사용중                                                                                     3000.01.08 \~ 3000.01.10                                                                             3000.01.15 \~ 3000.01.15 회의실 6C  6명                  2층            예약있음                                                                                3000.01.13 \~ 3000.01.13                                                                             3000.01.14 \~ 3000.01.14                                                                             3000.01.15 \~ 3000.01.20                   계속하려면 Enter 를 누르세요… |
| :---- |

예를 들어 위 출력에 따르면 ‘회의실 6B’ 의 현황은 \<사용중\>이므로, 현재 운영 시점은 ’회의실 6B’의   
예약일 기준 첫 번째 예약일인 3000.01.08 \~ 3000.01.10 기간에 포함된다.

모든 정보를 출력한 후 관리자가 엔터키를 포함한 문자들을 입력하면 관리자 메뉴로 복귀한다.

### **6.6.1.2 회의실 목록 조회 및 상태 변경**

관리자가 로그인 후 관리자 메뉴에서 메뉴 번호 2를 입력하고 엔터키를 누른 경우 회의실 정보를 출력한다. 출력하는 내용은 순서대로 선택 번호, 회의실 이름, 수용 인원, 위치, 상태다.  
아래 정보를 출력한 후 관리자가 상태 변경할 회의실 선택 번호 입력을 받는다.  
이때 상태는 사용가능, 점검중, 사용불가 총 3가지가 있다.   
사용불가는 당일 예약이 있는 경우에 출력된다. 사용할 수 있는 회의실인 경우 사용 가능, 점검 중인 회의실인 경우 점검중으로 출력된다.

| \================================================= 회의실 목록 \================================================= 번호       이름            수용인원           위치          상태 —--—------------------------------------------------------  1          회의실 4A    4명                  1층            사용가능   2         회의실 4B    4명                  1층            사용가능   3         회의실 4C    4명                  1층            사용가능  4         회의실 6A    6명                  2층            사용가능  5         회의실 6B    6명                  2층            사용가능  6         회의실 6C    6명                  2층            사용가능 0 : 취소 회의실 선택(번호): |
| :---- |

0 을 기입한 후 엔터키를 누르면 관리자 메뉴로 돌아가며, 그외 기입한 번호가 적합한 선택 번호일 경우 상태 번호를 기입해야 하는 다음 단계로 넘어갈 수 있다.   
적합한 선택 번호란, 출력되는 정보 중 회의실 선택 번호로 제공되는 번호(1\~N)를 뜻한다. 이때 제공되는 회의실 선택 번호의 마지막인 N 은 전체 회의실 수와 일치한다.  
즉, 위의 출력에 따르면 현재 N 은 6이자 전체 회의실 수가 6이라는 의미다.

문법 형식: 올바른  내용은 아래 조건을 모두 만족하는 문자열이다.

* 0 또는 1 이상 N 이하의 십진 정수  
* 공백 문자, 탭 문자만으로 구성되지 않음

위 문법 형식에 부합하지 않거나 전체 회의실 수와 동일한 값인 N 보다 큰 십진 정수를 기입한 경우, ‘1\~N 사이의 번호를 입력해주세요.’ 문구와 함께 다시 기입할 수 있는 기회를 제공한다.

| 회의실 선택 (번호): 999 1 \~ 9 사이의 번호를 입력해주세요. |
| :---- |

위 문법 형식에 부합하지 않은 십진 정수 외의 문자가 포함된 경우 ‘숫자를 입력해주세요.’ 문구와 함께 다시 기입할 수 있는 기회를 제공한다.

| 회의실 선택 (번호): hello9 숫자를 입력해주세요. |
| :---- |

	  
위 입력에서 관리자가 적합한 선택 번호를 기입한 경우 관리자에게 선택 가능한 회의실 상태를 출력하고 입력을 받는다.   
선택 가능한 상태 번호는 1, 2 각각 사용가능, 점검중에 해당한다.  
문법 형식:

* 1, 2 중 하나인 십진 정수  
* 공백 문자, 탭 문자만으로 구성되지 않음

| 변경할 상태 :  1\. 사용가능 (available) 2\. 점검중 (maintenance) 선택 :  |
| :---- |

위 문법 형식에 부합하지 않은 문자열을 기입한 경우 다음과 같이 출력하고 ~~관리자 메뉴로 복귀한다~~ 재입력을 받는다.

| :---- |

문법 형식에 부합한 문자열을 기입한 경우 다음과 같이 출력하며 입력을 받는다.

* ID: 가입 시 등록한 로그인 아이디를 출력하며, 중복되지 않는 고유 식별자로 활용한다.  
* 역할: 해당 계정의 권한을 `[관리자]` 또는 `[일반]` 으로 표시한다.  
* 패널티: 현재 누적된 패널티 점수를 출력한다.  
* 상태: 패널티 점수 및 운영 시계에 따른 현재 이용 가능 상태를 `정상`, `제한`, `금지` 등으로 실시간 판별하여 출력한다.

위 정보를 출력한 후 사용자가 엔터키를 입력하면 관리자 메뉴로 돌아간다.

### **6.6.3.2 사용자 상세 조회**

관리자가 로그인 후 메뉴 번호 14를 입력했을 경우, 다음과 같이 특정 사용자의 상세 정보를 조회하고 관리할 수 있는 기능을 제공한다.

출력하는 내용은 순서대로 선택 번호, 아이디, 역할이다.

| \==================================================  사 용 자 상 세 조 회 \==================================================  1\. admin \[관리자\]  2\. testuser \[일반\]  3\. damage\_user \[일반\]  4\. damage\_admin \[관리자\]  5\. range\_user \[일반\]  6\. range\_admin \[관리자\] … 사용자 선택 (번호): |
| :---- |

* 선택 번호는 관리자가 상세 조회할 유저를 쉽게 지정할 수 있도록 1번부터 순차적으로 부여된 번호를 의미한다.  
* 사용자명은 등록된 사용자의 아이디를 출력하여 관리자가 대상을 식별할 수 있게 한다.  
* 역할은 해당 사용자가 `[일반]` 유저인지 `[관리자]` 계정인지 표시한다.

관리자가 목록에서 특정 사용자의 선택 번호를 입력하면 해당 사용자의 상세 데이터가 출력되며, 이 과정에서 발생할 수 있는 예외 상황은 다음과 같이 처리한다.

* 범위를 벗어난 번호 입력: 출력된 리스트의 최대 번호(N)보다 큰 숫자나 0 이하의 숫자를 입력할 경우, `“1~N 사이의 번호를 입력해주세요.”` 메시지를 출력하고 다시 번호를 입력받는다.  
* 숫자가 아닌 문자 입력: 영문, 한글, 특수문자 등 정수가 아닌 값을 입력할 경우, `“숫자를 입력해주세요.”` 메시지를 출력하고 다시 번호를 입력받는다.  
* 빈 값 입력: 아무런 값도 입력하지 않고 엔터 키만 누를 경우, 현재 조회 대기 화면을 유지하며 관리자가 번호를 입력할 때까지 재입력을 요구한다.  
* 공백 포함 입력: 숫자 앞뒤에 공백이 포함된 경우, 시스템은 공백을 자동으로 제거(trim)하여 정상적인 숫자로 인식하도록 처리한다.

사용자 선택 번호를 입력한 후 출력되는 상세 정보는 사용자 기본 정보, 패널티 상태, 활성 예약, 패널티 이력으로 구성된다.

| ID : testuser1 역할 : \[일반\] 가입일 : 2027-06-15 10:00 \- \- \- 패널티 상태 \- \- \- 상태 : ✅ 양호 누적 점수 : 0점 정상 이용 연속 : 0회 \- \- \- 활성 예약 \- \- \- 회의실 : 0건 장비 : 0건 \- \- \- 패널티 이력 \- \- \- \- 2026-03-20 18:00: late\_return (+2점) 회의실 반납 지연 계속하려면 Enter를 누르세요... |
| :---- |

* 사용자 기본 정보는 선택된 사용자의 아이디와 계정 역할(`[일반]`/`[관리자]`), 그리고 시스템에 가입한 구체적인 날짜와 시각 정보를 제공한다.  
* 패널티 상태는 현재 유저의 상태를 누적된 전체 점수와 함께 패널티 감면에 이용 되는 정상 이용 연속 횟수를 실시간으로 합산하여 보여준다.  
* 활성 예약은 해당 사용자가 현재 이용 중인 회의실과 장비의 건수를 출력하며, 구체적인 회의실 명칭과 이용 기간(시작 시각 \~ 종료 시각)을 상세히 나열한다.  
* 패널티 이력은 과거에 발생한 위반 사항을 역순으로 출력하며, 위반 발생 시점과 사유를 보여준다.

모든 정보를 확인한 후 사용자가 엔터키를 입력하면 다시 관리자 메뉴로 돌아간다.

### **6.6.3.3 패널티 정책**

#### **6.6.3.3.1 패널티 상태 기준**

패널티는 점수 형태로 누적 관리된다.

누적 패널티 점수가 0점 이상 2점 이하인 경우 사용자는 정상(normal) 상태이다.

정상 상태의 사용자는 회의실 활성 예약 1건, 장비 활성 예약 1건까지 유지할 수 있다.

누적 패널티 점수가 3점 이상 5점 이하인 경우 사용자는 제한(restricted) 상태가 된다.

제한 상태의 사용자는 회의실 예약과 장비 예약 활성 예약을 각각 1건까지만 유지할 수 있다.

이미 활성 예약이 1건 존재하는 경우 추가 예약 생성은 허용되지 않는다.

누적 패널티 점수가 6점 이상인 경우 사용자는 이용금지(banned) 상태가 된다.

이용금지 상태의 사용자는 새로운 예약을 생성할 수 없다.

또한 현재 운영 시점 이후에 시작하는 예약은 자동으로 취소된다.

제한 또는 이용금지 상태가 부여되면 시스템은 해당 상태의 해제 예정 시점을 기록한다.

운영 시계 진행 중 해제 예정 시점이 현재 운영 시점 이하가 되면 해당 제한 상태는 해제된다.

패널티가 부과되면 정상 이용 연속 횟수는 0으로 초기화된다.

#### **6.6.3.3.2 정상 이용보상**

사용자가 정상 이용을 10회 연속 달성한 경우 누적 패널티 점수는 1점 감소한다.

단, 누적 패널티 점수가 0점인 경우에는 추가 차감하지 않는다.

정상 이용은 다음 조건을 모두 만족한 예약 1건을 의미한다.

\- 회의실 예약의 경우 체크인 승인 후 정상 퇴실 승인까지 완료된 경우

\- 장비 예약의 경우 픽업 승인 후 정상 반납 승인까지 완료된 경우

\- 이용 과정에서 별도의 패널티가 부과되지 않은 경우

\- 사용자 취소, 노쇼, 지연, 파손, 오염, 관리자 강제 취소가 발생하지 않은 경우

정상 이용 보상이 적용된 경우 연속 정상 이용 횟수는 다시 0으로 초기화된다.

보상 적용 결과 누적 패널티 점수가 제한 기준 미만이 되면 제한 상태는 해제될 수 있다.

#### **6.6.3.3.3 패널티 초기화**

사용자의 마지막 패널티 부과 시점으로부터 90일 이상 추가 패널티 없이 경과한 경우,

누적 패널티 점수는 0점으로 초기화된다.

해당 초기화는 운영 시계 진행 과정에서 시스템에 의해 자동으로 반영된다.

초기화가 수행되면 기존에 설정된 해제 예정 시점이 존재하는 경우 이를 함께 제거할 수 있다.

패널티 이력 자체는 삭제하지 않으며, 현재 사용자 상태 판단에만 초기화 결과를 반영한다.

#### **6.6.3.3.4 패널티 부과 공통 원칙**

패널티는 자동 부과와 수동 부과로 구분된다.

자동 부과는 적용 기준이 명확한 경우, 운영 시점 전환 또는 관련 처리 과정에서 시스템에 의해 자동으로 기록된다.

자동 부과 대상에는 직전 취소, 퇴실 지연, 반납 지연이 포함된다.

수동 부과는 파손, 오염 등 관리상 필요에 따라 관리자가 직접 부과하는 방식이다.

수동 부과 시에는 반드시 관련 예약 또는 대상 자원을 확인할 수 있어야 하며, 부과 사유를 입력하여 기록으로 남겨야 한다.

동일한 사건에 대해 동일한 의미의 패널티를 중복 부과할 수 없다.

예를 들어 동일한 예약 건에 대해 이미 무단 미사용 패널티가 부여된 경우 동일 사유의 패널티를 다시 부과할 수 없다.

또한 이미 반납 지연 패널티가 부여된 예약에 대해 동일 사유의 패널티를 중복 기록할 수 없다.

### **6.6.3.4 파손/오염 패널티 부여**

관리자가 로그인 후 메뉴 번호 15을 입력했을 경우 시설물을 파손하거나 오염시킨 사용자에게 패널티를 부여한다. 

먼저 현재 시스템에 등록된 유저중 예약 이력이 있거나 예약 진행 중인 유저들을 번호와 함께 출력하여 관리자가 패널티를 부여할 대상을 선택할 수 있게 한다.

| \==================================================   파 손 / 오 염   패 널 티   부 여 \==================================================  1\. testuser  2\. damage\_user  3\. range\_user  4\. maint\_user  5\. disabled\_user … 0\. 취소 사용자 선택 (번호): |
| :---- |

사용자 선택 단계:

* 사용자 선택 입력란에 숫자 이외의 문자를 입력할 경우, 시스템은 “숫자를 입력해 주세요."라는 안내 메시지를 출력하고 올바른 형식이 입력될 때까지 재입력을 요구한다.  
* 현재 화면에 표시된 사용자 번호의 범위를 벗어난 숫자를 입력할 경우, 시스템은 "1\~N 사이의 번호를 입력해 주세요."라는 메시지를 통해 유효한 범위를 명시적으로 안내한다.  
* 0번(취소)을 입력하면 패널티 부여 절차를 중단하고 상위 관리자 메뉴로 복귀한다.

예약 유형 선택 단계:

* 시스템은 항상 ‘1. 회의실 예약’, ‘2. 장비 예약’ 두 가지 선택지를 고정으로 출력합니다.  
* 1 또는 2 이외의 숫자나 문자가 입력될 경우, “x 잘못된 선택입니다.” 메시지를 출력합니다. 

관련 예약 선택 단계:

* 관련 예약 선택 입력란에 숫자 이외의 문자를 입력할 경우, 시스템은 “숫자를 입력해 주세요."라는 안내 메시지를 출력하고 올바른 형식이 입력될 때까지 재입력을 요구한다.  
* 현재 화면에 표시된 예약 번호의 범위를 벗어난 숫자를 입력할 경우, 시스템은 "1\~N 사이의 번호를 입력해 주세요."라는 메시지를 통해 유효한 범위를 명시적으로 안내한다.  
* 0번(취소)을 입력하면 패널티 부여 절차를 중단하고 상위 관리자 메뉴로 복귀한다.

패널티 점수 입력 단계:

* 패널티 점수 입력 시 숫자 형식이 아닌 문자를 입력할 경우, 시스템은 “x 숫자를 입력해주세요.” 라는 경고 메시지를 출력하며 재입력을 요구한다.  
* 규정된 범위(1\~5점)를 벗어난 숫자를 입력할 경우에도 올바른 범위 내의 값을 입력할 때까지 반복적으로 재입력을 유도한다.

패널티 부과 세부 기준:

* 경미한 장비 파손 (1점): 외관 스크래치나 마우스 패드 훼손 등 기능상 문제가 없는 단순 파손의 경우 부과한다.  
* 시설 및 장비 오염 (1점): 회의실 내 음식물 반입이나 바닥/책상의 심한 오염으로 별도의 청소가 필요한 경우 부과한다.  
* 구성품 분실 (2점): 대여 장비의 전용 어댑터, 마우스, 스타일러스 펜 등 부속품을 분실했을 경우 부과한다.  
* 심각한 장비 파손 (4점): 노트북, 태블릿 파손 등 수리가 필수적인 기능 고장이 발생했을 경우 부과한다.  
* 고정 시설물 파손 (4점): 회의실 의자, 빔프로젝터, 벽면 마감재 등 고정 자산을 훼손했을 경우 부과한다.

#### 입력 및 승인 규칙:

* 사유 입력: ~~패널티 점수 입력 후 해당 위반 건에 대한 구체적인 사유를 서술형으로 입력받아 로그에 기록한다.~~  사유 작성시 5.5절의 사유 문법 규칙에 따른다.

* 최종 확인 (y/n): 관리자에게 "{사용자명}에게 2점 패널티를 부여하시겠습니까?"라는 메시지를 출력하여 최종 의사를 확인한다. 이때 승인 확인 입력으로 y, yes, 예, ㅇ(대소문자 무관)를 입력하면 실제 데이터에 반영되어 패널티가 부과된다. 반면 n, no, 아니오, ㄴ을 입력하면 패널티 부여를 철회하고 관리자 메뉴로 돌아간다. 그 외의 값을 입력할 경우 "y 또는 n을 입력해주세요."를 출력하고 올바른 값이 입력될 때까지 재입력을 유도한다.  
* 상태 자동 갱신: 패널티 부여 결과 사용자의 누적 점수가 기준치를 초과하면 시스템은 즉시 해당 사용자의 이용 상태를 전이시키고 이를 화면에 안내한다.

| 사용자 선택 (번호) : 130 예약 유형 :  1\. 회의실 예약  2\. 장비 예약 선택 : 1  1\. 4A \- 2027-06-15 11:00\~12:00   2\. 6B \- 2027-06-15 13:00\~14:00   0\. 취소 관련 예약 선택 (번호) : 1  ~~사유 : 직전 취소~~  testuser1에게 2점 패널티를 부여하시겠습니까? (y/n) : y ✓ 패널티가 부여되었습니다. (+2점) i 사용자 현재 누적 : 3점 ⚠️ 사용자가 예약 제한 상태가 되었습니다. 계속하려면 Enter를 누르세요... |
